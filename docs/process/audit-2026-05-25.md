# Cage Rentals — deep-sweep audit (2026-05-25)

Two parallel sweeps across UI/aesthetic + engineering quality. Findings consolidated by priority, with suggested handoff batches at the bottom.

## TL;DR — top 3 things to fix this week

1. **Snapshot rule violated on 4 admin surfaces (real money bug).** Reports, Excel export, and `/admin/sessions` correctly read `sessionsBilling.ratePer30MinCents`. But `/admin`, `/admin/payments`, `/admin/coaches`, `/admin/coaches/[id]` all silently *recompute* from current overrides. The moment Dad renegotiates a rate, these pages diverge from the report. `/admin/payments` is what Dad uses to chase money — so this surfaces as "the dashboard says I'm owed X but the report says I'm owed Y."
2. **No global error/404 pages.** Any unhandled error shows stock React red overlay (dev) or blank page (prod). One-evening fix.
3. **Pervasive raw-Tailwind colors that violate the design spec.** `emerald`/`sky`/`amber`/`red` used directly instead of `success`/`warning`/`danger`/`gold` tokens. Looks fine today but breaks the moment we tweak a token.

Plus a category of gap: **test coverage** outside `billing.ts` is thin. Payment-actions, rate overrides, merges, deletes — none of the critical mutation paths have tests.

## What's genuinely well-built (worth celebrating)

- **`src/lib/timezone.ts` + `src/lib/billing.ts`** — pure, well-tested, principled. The bones are right.
- **The `internal action + thin public wrapper` pattern** held consistently across 9 server-action files. Only one violation found (E5).
- **Token system in `globals.css`** is well-documented and includes a deliberate WCAG fix for `fg-subtle` (3.78 → 5.32 contrast). The system is good; usage just drifts.
- **`delete-coach-card.tsx`'s typed-confirmation modal** is the right pattern — should become system-wide `<ConfirmDialog>`.
- **`src/app/_components/skeleton.tsx`** — hand-rolled with proper `role="status"` + `sr-only` "Loading…". Better than a shadcn import would be.
- **Admin dashboard stat-tile + NavCard composition** in `admin/page.tsx` is genuinely good information design.
- **DB constraints** (EXCLUDE + CHECK on `sessions_billing` and `blocked_times`) are doing real work; app-layer overlap checks are belt-and-suspenders.
- **Soft-delete + merge flows** are carefully thought through (anonymization, email TLD swap, session/account cleanup).
- **Architecture + runbook docs** are unusually thorough and current.
- **Consistent eyebrow / page-header / back-link pattern** across admin pages.

## What I'm NOT recommending

- **Don't centralize `safeLogAudit` yet (E18).** It's duplicated but identical — a one-line abstraction. Wait until something diverges before pulling.
- **Don't fight neon-http's no-transactions (E19, E20).** Documented. The recovery paths exist. Migrate to neon-serverless only when compliance forces it.
- **Don't remove `allowDangerousEmailAccountLinking` (E12).** Threat model is sound; the comment justifies it.
